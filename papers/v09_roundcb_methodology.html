Methodology · v09 RoundCB

Federated Residual Codebook on a Co-trained NBEATSx Backbone

NBEATSx 잔차 추출 → 2-stage federated 집계(Aggregation) → 사후 보정(Correction)의 전 과정. 근거: 03_fl_per_round_codebook.py, src/fl/codebook_fl.py, src/models/nbeatsx.py
설계 요지. RoundCB는 세 모듈로 구성된다 — (R) NBEATSx 잔차 분해로 표현 추출, (A) 표현에 대한 2-stage federated 군집화(codebook), (C) 군집별 평균 forecast 잔차(offset)로 예측 보정. 핵심은 두 종류의 잔차를 구분하는 것이다: (i) 입력측 backcast 잔차(trend·seasonal이 설명 못 한 부분 → generic hidden hg로 인코딩)와 (ii) 출력측 forecast 잔차 y−ŷ(codebook offset으로 저장). codebook은 (i)로 라우팅하고 (ii)로 보정한다. codebook은 backbone forward를 거치지 않으므로 표현 형성(R-loop)은 NBEATSx 자체에 국한된다.

M.1 표기 · 데이터

데이터UMass Smart* 2016 시간별 부하, filter_valid_apartments(min_hours=7000) 통과 가구 N=114. 가구당 시계열을 시간 순서로 train 70% / val 10% / test 20% 분할(stride=24, 비중첩 sliding window).
정규화가구별 z-norm (μa, σa)는 train 구간에서만 추정(invariant).
윈도입력 x ∈ ℝL, L=INPUT_SIZE=96 → 예측 ŷ ∈ ℝH, H=HORIZON=24.
하이퍼파라미터D_MODEL=64, codebook 크기 M=32, 가구별 Stage-1 군집 수 Klocal=2, 보정 강도 αv0=1.0, 보조 손실 λaux=0.3 (hr_weight=0.1). 라운드 R, 로컬 epoch E.
클라이언트가구 i ∈ {1..114}, 각자 윈도 집합 𝒟itrain. 전원 참여(C=1.0).

RM.2 NBEATSx 잔차 분해 — 표현 hg 추출

backbone은 NBEATSxAux(latent_source='h_generic') — 3-stack MinimalNBEATSx(trend / seasonal / generic) 본체에 PeakAuxHead를 붙인 형태다. NBEATS의 핵심은 iterative residual decomposition: 각 stack이 입력의 일부를 설명(backcast)하고, 남은 잔차를 다음 stack으로 넘긴다.

x ∈ ℝ⁹⁶ (z-normed) │ r⁽⁰⁾ = x ▼ ┌─ Trend stack ──────────────┐ 4-layer MLP(fc1..fc4) → hₜ │ (bₜ, fₜ, hₜ) = Stack_t(r⁽⁰⁾) │ θ=proj(h) → Polynomial basis └──────────────┬───────────────┘ r⁽¹⁾ = r⁽⁰⁾ − bₜ (trend 제거 후 잔차) ▼ ┌─ Seasonal stack ───────────┐ → h_s, Harmonic basis │ (b_s, f_s, h_s)=Stack_s(r⁽¹⁾)│ └──────────────┬───────────────┘ r⁽²⁾ = r⁽¹⁾ − b_s ← trend·seasonal이 설명 못 한 잔차 ▼ ┌─ Generic stack ────────────┐ Identity basis │ (b_g, f_g, h_g)=Stack_g(r⁽²⁾)│ h_g = fc4(...) ∈ ℝ⁶⁴ └──────────────────────────────┘ ↑ "잔차의 latent 요약" │ ▼ ŷ = fₜ + f_s + f_g ∈ ℝ²⁴ h_g → (PeakAuxHead, codebook 라우팅)

각 stack의 내부 (nbeatsx.py:95) 와 분해 (nbeatsx.py:139):

stack 내부: h = fc₄( σ( fc₃( σ( fc₂( σ( fc₁(r) ) ) ) ) ) ) σ = ReLU, h ∈ ℝ⁶⁴ θ = proj(h); (backcast, forecast) = Basis(θ)
r⁽⁰⁾ = x (bₜ, fₜ, hₜ) = Stack_t(r⁽⁰⁾); r⁽¹⁾ = r⁽⁰⁾ − bₜ (b_s, f_s, h_s) = Stack_s(r⁽¹⁾); r⁽²⁾ = r⁽¹⁾ − b_s (b_g, f_g, h_g) = Stack_g(r⁽²⁾) generic은 정제된 잔차 r⁽²⁾ 위에서 동작 ŷ = fₜ + f_s + f_g
왜 hg가 "잔차 표현"인가. generic stack은 trend·seasonal backcast를 제거한 이중 정제 잔차 r⁽²⁾를 입력으로 받는다. 따라서 그 hidden hg=fc₄(·) ∈ ℝ⁶⁴는 "공통 구조(추세·계절성)로 설명되지 않은, 가구별·희귀 패턴"을 압축한 latent다. RoundCB는 codebook을 바로 이 hg 위에 세운다 — 보내주신 "residual → prototype memory"의 residual이 코드에서 정확히 hg에 대응한다.

M.3 Federated backbone 학습 (codebook과 분리)

backbone은 run_fl_aux로 표준 federated 학습된다(알고리즘 ∈ {FedAvg, FedProx, FedRep, Ditto, FedProto}, 기본 FedAvg). 라운드마다 server state를 broadcast → 각 가구 E epoch 로컬 학습 → 집계. 로컬 손실:

𝓛 = MAE(ŷ, y) + λaux · peak_aux(â, ĥ, y; hr_weight) λ_aux=0.3, hr_weight=0.1

codebook은 이 손실에 들어가지 않는다. 즉 backbone은 codebook의 존재를 모른 채 학습되며, 표현 형성(R-loop)은 NBEATSx 잔차 분해 + peak-aux head로만 일어난다. 이것이 RoundCB가 post-hoc인 이유다 (03 docstring: forward가 codebook을 절대 통과하지 않음, no commit_loss).

AM.4 라운드별 2-stage Federated Codebook (Aggregation)

매 라운드 종료 시(주기 codebook_period), 그 시점의 frozen backbone 위에서 codebook을 처음부터 다시 fit한다. 세 단계 모두 federation contract를 지킨다 — raw hg·raw 잔차는 가구를 벗어나지 않고, 요약 통계만 업로드된다.

Stage 1 — 가구 로컬 KMeans (codebook_fl.py:482 local_codebook_step_from_splits)

가구 i가 자신의 train 윈도를 frozen forward → Hi = { hg(x) : x ∈ 𝒟itrain } ∈ ℝNi×64. 그 위에 로컬 KMeans++(Klocal=2):

{ μi,k, ni,k }k=1..Ki = KMeans++(Hi, Klocal) μ: centroid ∈ ℝ⁶⁴, n: count, Σ n = Ni 업로드 = (centroids μi,·, counts ni,·) raw h_g는 업로드 안 함

소형 가구 fallback: Ki = min(Klocal, Ni).

Stage 2 — 서버 가중 KMeans (codebook_fl.py:236 merge_local_codebooks)

서버가 모든 가구의 centroid를 쌓고, 각 centroid를 그 count로 가중하여 전역 M개로 재군집:

P = ⋃i { μi,k }, w = ⋃i { ni,k } C = KMeans++(P, M=32; sample_weight = w) ∈ ℝ32×64 전역 codebook (broadcast)

진단치도 함께 계산: utilization(비어있지 않은 cluster 비율), perplexity = exp(entropy), k_min/k_max, Stage-2 inertia.

Stage 3 — Federated 잔차 offset (codebook_fl.py:351 federated_residual_offsets)

각 가구가 자신의 hg를 전역 codebook에 1-NN 라우팅하고, cluster별 forecast 잔차 부분합과 count를 업로드. 서버가 cluster별로 평균:

c*i(x) = argminm ‖ hg(x) − Cm ‖₂ 입력측 잔차로 라우팅 ri(x) = yz(x) − ŷz(x) ∈ ℝ²⁴ 출력측 forecast 잔차 업로드 = ( Σx:c*=m ri(x), #{x:c*i=m} )m 부분합·count만 서버: om = ( Σi Σx:c*i=m ri(x) ) / max( Σi #{x:c*i=m}, 1 ) ∈ ℝ²⁴

결과 offsets ∈ ℝ32×24. 비어있는 cluster는 offset=0 (그 경로의 ŷ는 무보정). 이 om이 곧 "cluster m으로 라우팅되는 패턴들의 전형적 예측 오차" — prototype memory의 저장 내용물이다.

CM.5 테스트 시 보정 (Correction)

가구의 test 윈도를 frozen backbone에 1회 forward → (hgtest, ŷbase,z). codebook 라우팅 후 해당 offset을 더하고 kW로 역정규화 (03:255-263):

c* = argminm ‖ hgtest − Cm ‖₂ ŷcorr,z = ŷbase,z + αv0 · oc* αv0 = 1.0 (carry-over, 재튜닝 안 함) ŷkW = ŷcorr,z · σa + μa 가구별 z-norm 역변환

지표는 kW 공간에서 가구별 PAPE / HR@1 / HR@2 / MAE / MSE(kW²)를 산출 후 가구 평균(±std, ddof=1). 각 라운드마다 보정 전(test_before)·후(test_after)와 그 차이(lift ΔPAPE 등)를 JSONL 1행으로 기록 → per-round lift trajectory.

M.6 전체 파이프라인 한 장

══ 매 라운드 r = 1..R ══════════════════════════════════════ [Server] broadcast backbone state θ_r │ ▼ 각 가구 i: E epoch 로컬 학습 [Client i] 𝓛 = MAE(ŷ,y) + 0.3·peak_aux ← codebook 미관여 │ upload Δθ_i ▼ [Server] θ_{r+1} = FedAvg({θ_i}) (R: backbone 표현) ── 라운드 끝: codebook fit (frozen θ_{r+1}) ────────────── [Client i] h_g(𝒟_i) → Stage1: KMeans(K=2) → (μ_i, n_i) ↑ [Server] Stage2: weighted KMeans(M=32) → C ↓ broadcast (A) [Client i] route h_g→C, r=y−ŷ → (Σr_m, cnt_m) ↑ [Server] Stage3: o_m = Σr_m / Σcnt_m → offsets (A) │ ▼ test 윈도 평가 [Eval] ŷ_corr = ŷ_base + α·o_{c*(h_g^test)} (C) → before/after PAPE, lift → codebook_log.jsonl[r] ════════════════════════════════════════════════════════════
Algorithm — RoundCB (per-round federated residual codebook)
input: clients {𝒟_i}, rounds R, local epochs E, M=32, K=2, α=1.0
init : backbone θ ← init_backbone_aux(seed)
for r = 1 .. R:
    # ---- backbone (codebook-agnostic) ----
    for each client i:  θ_i ← LocalTrain(θ, 𝒟_i, E)   # 𝓛=MAE+0.3·aux
    θ ← FedAvg({θ_i})
    # ---- Stage 1: local KMeans on residual rep h_g ----
    for each client i:
        H_i ← h_g( forward_frozen(θ, 𝒟_i^train) )
        (μ_i, n_i) ← KMeans++(H_i, K)               # upload (μ,n) only
    # ---- Stage 2: server weighted merge ----
    C ← KMeans++( stack{μ_i}, M; sample_weight=stack{n_i} )
    # ---- Stage 3: federated residual offsets ----
    for each client i:
        c*_i ← 1NN(H_i, C);  r_i ← y_z − ŷ_z
        upload ( Σ_{c*=m} r_i ,  count_m )
    o_m ← Σ_i Σ r_i,m / max(Σ_i count_i,m, 1)        # offsets ∈ ℝ^{M×H}
    # ---- Correction + log ----
    for each client i:  ŷ_corr ← ŷ_base + α·o_{1NN(h_g^test, C)}
    log[r] ← { before, after, lift=after−before, util, perplexity }
return θ, {C_r, o_r}_r, lift_trajectory

M.7 Federation contract · 산출물

프라이버시 계약

  • Stage 1: raw hg 대신 (centroid, count)만 업로드.
  • Stage 3: raw 잔차 대신 (잔차 부분합, count)만 업로드.
  • 서버는 가구별 윈도를 절대 보지 않음. v05 FedCB 계약을 그대로 계승.

per-seed 산출물

  • codebook_log.jsonl — 라운드별 before/after 7-axis + lift + 진단.
  • codebook_history.pt — {rounds, codebook (R',M,64), offsets (R',M,24)}.
  • result.json — terminal 요약 + lift_trajectory.
R/A/C 위치 재확인. RoundCB는 A·C만 구동한다. (R)은 NBEATSx 잔차 분해 + peak-aux가 담당하며 codebook은 표현에 되먹이지 않는다. codebook이 표현 형성에까지 관여하는 통합 버전(commitment loss로 R까지 켜는 것)은 별도 driver 02_fl_vq_dynamics.py (FedVQ)이며, RoundCB의 유일한 출력 영향 경로는 offset 보정(C)이다.
